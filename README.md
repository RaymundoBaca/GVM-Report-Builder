# GVM-Report-Builder
Generate editable DOCX/XLSX/TXT reports from GVM (Greenbone/OpenVAS) TXT+PDF outputs.

[![Python](https://img.shields.io/badge/Python-3.10%2B-blue.svg)]()
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Build with PyInstaller](https://img.shields.io/badge/Build-PyInstaller-green)]()

Turn messy **GVM/OpenVAS vulnerability scan outputs** into **professional reports** (Word, Excel, TXT) that are **well-structured, editable, and client-ready**.  

---

## ✨ Features
- 📑 Parse **TXT + PDF reports** generated by **GVM/OpenVAS**.  
- 📝 Create a clean and simplified `Summary.txt`.  
- 📊 Generate:
  - `Report.docx` → Word file with all vulnerabilities, CVSS scores, mitigation tips and screenshots.  
  - `TableFinal.xlsx` → Excel sheet with tabular view of vulnerabilities.  
  - `Summary.txt` → Flat text file with cleaner structure.  
- 🖼️ Attach vulnerability screenshots automatically (based on naming convention).  
- ⚡ Easy-to-use GUI (PySide6) with one-click generation.  
- 🔎 Integrated **Tesseract OCR** for image text extraction (already packaged in `.exe`, no installation needed).  

---

## 🚀 Quickstart

### 1. Clone the repo

git clone https://github.com/RaymundoBaca/GVM-Report-Builder.git
cd gvm-report-builder

### 2. Install dependencies

pip install -r requirements.txt

### 3. Run the tool
python Vuln.py

## Usage Guide

### 📂 Input Files
The program requires **two input files** exported from the GVM panel by the user:
1. A PDF report  
2. A TXT report  

When you run the program, it will ask you for an **output folder**, where it will automatically create a `Results/` folder. Inside `Results/`, the tool will generate the following files:

Results/
├─ ReporteVuln.docx    # Word report with all vulnerabilities, CVSS scores, mitigation tips, and screenshots
├─ TableFinal.xlsx      # Excel table with vulnerabilities organized by host
└─ Summary.txt          # Simplified text summary
